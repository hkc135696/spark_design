# 智慧交通实时路况监测与拥堵分析系统 - 开发文档

> 本文档面向后续开发人员，提供项目全景概览、开发进度追踪与继续开发指南。

---

## 一、项目概述

本项目是一套基于 **Python + Kafka + Spark Structured Streaming + MySQL + Flask + Vue 3** 的智慧交通实时流处理系统。模拟北京市 105 个交通检测器实时采集路况数据，经流处理后输出多维分析结果，通过 Web 前端可视化展示。

---

## 七、Spark 作业详细说明

### 7.1 数据清洗作业 `spark_cleaner.py`

**输入**: `traffic-raw-data` Topic（10 秒触发）\
**输出**:
- MySQL `traffic_raw` 表（pymysql 直写，append 模式）
- Kafka `traffic-cleaned` Topic（confluent-kafka Producer）

**清洗规则**:
- `COALESCE(detector_id, '')` → 空值过滤
- `CASE WHEN avg_speed < 0 OR avg_speed > 200 THEN NULL` → 速度越界
- `CASE WHEN occupancy < 0 OR occupancy > 100 THEN NULL` → 占用率校验
- `CASE WHEN congestion_index < 0 OR congestion_index > 10 THEN NULL` → 拥堵指数范围
- `dropDuplicates(['detector_id', 'timestamp'])` → 去重

**输出字段**: 原始字段 + `date_key` (DATE) + `hour_key` (VARCHAR) + `etl_time` (TIMESTAMP)

**指标输出**: `cleaner_metrics.json` (总数/过滤数/去重数/入库数), `cleaner_errors.json` (错误详情)

**状态管理**: 使用 RocksDBStateStoreProvider 存储累计进度，支持故障恢复

**启动命令**:
```powershell
python spark_streaming\data_cleaning\spark_cleaner.py
```

---

### 7.2 基础分析作业（5 个）

均消费 `traffic-cleaned` Topic，30 秒触发，写入 MySQL（upsert via `REPLACE INTO`）。

| 作业 | 聚合维度 | 关键指标 | 输出表 |
|------|---------|---------|--------|
| `①_road_flow.py` | detector_id + 1min 窗口 | SUM(total_flow), AVG/MAX/MIN(avg_speed), AVG/MAX(congestion), AVG(occupancy) | `road_stats_realtime` |
| `②_avg_speed.py` | detector_id | AVG, STDDEV, MAX, MIN, PERCENTILE_APPROX(median) | `speed_stats` |
| `③_congestion_index.py` | detector_id | 复合公式: `LEAST(1.0, (flow/200) * 道路系数 * (0.5 + (1 - speed/80)*0.5))` | `congestion_index` |
| `④_peak_traffic.py` | district + date + hour | SUM(flow), AVG(congestion), 分类 peak_type | `region_hourly_stats` |
| `⑤_region_heat.py` | district | 热度评分: `flow/1000*0.3 + avg_congestion*0.5 + (100-speed)/100*0.2` | `region_heat_rank` |

**启动命令**:
```powershell
spark-submit spark_streaming\basic_analytics\①_road_flow.py
spark-submit spark_streaming\basic_analytics\②_avg_speed.py
spark-submit spark_streaming\basic_analytics\③_congestion_index.py
spark-submit spark_streaming\basic_analytics\④_peak_traffic.py
spark-submit spark_streaming\basic_analytics\⑤_region_heat.py
```

### 8.2 WebSocket 推送

每 **30 秒**推送一次，事件类型:
- `kpi_update`: 全局 KPI 数据（总处理量/过滤率/去重率等）
- `rank_update`: 区域排行 + 热点道路 Top10

### 8.3 作业管理

使用 `subprocess.Popen` + `psutil` 管理 13 个作业进程（含待开发作业占位）:

```python
# _JOBS 字典结构
{
    "simulator": {"name": "...", "command": "python ...traffic_simulator.py", "type": "python"},
    "cleaner":   {"name": "...", "command": "python ...spark_cleaner.py", "type": "python"},
    "road_flow": {"name": "...", "command": "spark-submit ...①_road_flow.py", "type": "spark"},
```
